learnt today:

- Traits as "contracts": They define what a struct can DO, not what it IS.
- Polymorphism with Traits: Using `<T: Trait>` or `&impl Trait` to make functions that work with multiple types (like `Animal` for Dog, Cat and Parrot).
- &impl Trait: Learned how to use references in traits to avoid "killing" (dropping) the object when calling a function.
- HashMaps (K, V): Storing key-value pairs. Used `entry(key).or_insert(0)` to build an efficient frequency counter.
- String vs &str (again): Managed the constant war between "photos" (&str) and "boxes" (String) when pushing data into collections.
- Ownership in collections: HashMaps and Vectors need to own their keys/values, so we used `.to_string()`.
- LinkedLists: Learned about double-linked chains. They are great for `push_front` and `push_back` operations (Spotify-style queue).
- Handling "Moved" values: Understood that passing a list to a function "gives it away", so we have to return it back (`list = function(list)`) or use mutable references.
